Solving the N body problem using Finite Difference Methods

The N-body problem is an initial value problem, whose solution is dictated by a governing differential equation and the initial conditions. The governing differential equation cannot be solved by simple algebra, so another method needs to be used to approximate the solution. In this project, we will explore the solution to an N-body problem using finite difference methods.

White-Box Identification of Transfer Function for Dynamics of a Small-Scale Quadcopter

White-box identification, a specific type of system identification, is a numerical scheme commonly applied in control systems to estimate unknown parameters associated with an assumed dynamical model. The initial goal of this project is to identify transfer functions that govern the input-output relationship between the torques, the net thrust, and all transnational and rotational degrees of free dome of the drone. The final result accomplished by the team is the identification of a transfer function that relates the net thrust to the vertical position of the drone. A PD controller is designed and tuned in MATLAB to stabilize the open-loop step response of the identified transfer function.

Differential Drive Robot

This report outlines my process of designing, implementing, and testing a controller I designed for a differential drive robot. The aim of the controller is to be able to move the robot around a circular path from start to finish without toppling over the edge as well as meeting certain requirements. In my process of designing a controller for this purpose, I found that there were certain limitations that were imposed on factors such as the forward speed, lateral error and others that caused me to try multiple iterations until I could find a suitable range of values for my controller to work properly. In this report I go through my process from start to finish including my explanation of the limits of my controller.

Controller-Observer implementation for Star-Tracker Spaceship

This report outlines my process of designing, implementing, and testing a controller I designed for a star-tracker spaceship. The aim of the controller is to be able to provide adequate torque to each of the wheels such that the spaceship is able to track the star (at random initial positions) throughout the simulation time period. To make this more challenging random "shooting stars" will interfere with the spaceship disturbing it’s orientation but the controller must be able to compensate for the change. In my process of designing a controller for this purpose, along with checking for controllability using the controllability matrix, I also had to use the observability matrix to check for observability. Unlike previous projects where we knew the state of the body exactly at all times, we don’t know the exact state solution this time and so we will have to work with state estimates from a sensor model.

Slingshot/Fly-By Simulation

I made this simulation to better understand slingshot trajectories and fly-bys around planetary bodies. When a spacecraft wants to maneauver around in space, sometimes it can leverage the gravitational pull of planets or other high-mass celestial bodies to change its attitude. The simulation allows the user to select parameters of the center body (Jupiter as a placeholder), the spacecraft's initial conditiond such as position and velocity to see if the spacecraft would be able to change trajectory as needed or will crash into Jupiter. At the espace velocity, the spacecraft can also maintain orbits around Jupiter
